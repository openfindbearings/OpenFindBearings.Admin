# 图片管理模块 v1.0.0

## 概述

图片管理模块用于人工修正爬虫抓取的轴承图片信息。包括图片缺失检测、图片错误更换、手动上传补全等功能。所有图片 URL 管理通过 OpenFindBearings.Api 的轴承编辑接口进行。

## 问题背景

CBIA 爬虫抓取的图片可能存在以下问题：
- 图片 URL 错误/失效（404）
- 图片内容与轴承型号不匹配
- 某些轴承缺少图片（Image1/Image2/3D/2D 缺失）
- 图片分辨率低或显示不全

## 图片存储策略

### 方案 A：URL 引用（推荐）

爬虫图片已有 URL（CBIA 服务器上的公开地址）。Admin 管理这些 URL 字段，不存储图片文件。仅当 URL 失效时，提供手动上传功能。

### 方案 B：本地存储

上传的图片存储到 Admin 的 `wwwroot/uploads/bearings/` 目录，然后更新轴承的 ImageUrl 字段。Admin 作为图片代理服务器（缓存的图片比 CBIA 源更快）。

**建议先用方案 A**，当发现大量图片失效时再切换到方案 B。

### 自动图片检测

在 Admin 后台可触发一次图片健康检查：

```
POST /Image/CheckHealth

对每个轴承的 ImageUrl 发送 HEAD 请求检查 HTTP 状态码
结果：{total: 11692, ok: 8500, notFound: 1500, error: 1692}
```

每日凌晨健康检查可整合到 ETL 流程中。
