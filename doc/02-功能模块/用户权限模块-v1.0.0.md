# 用户权限模块 v1.0.0

## 概述

用户权限模块通过 OpenFindBearings.Identity（基于 OpenIddict）实现统一认证和授权。用户在同一身份池中管理，Admin 提供用户管理、角色管理和权限分配功能。

## 认证方式

Admin 作为 OpenIddict 的 confidential client（机密客户端），使用 Authorization Code Flow + PKCE：

```
1. 用户访问 Admin /Account/Login
2. 重定向到 Identity /connect/authorize
3. 用户在 Identity 登录页输入用户名密码
4. Identity 验证后返回 authorization code
5. Admin 用 code + client_secret → 换取 access_token + refresh_token
6. Admin 将 token 存入加密 Cookie
7. 后续 API 调用携带 access_token
8. Token 过期 → 自动用 refresh_token 刷新
9. OAuth 退出 → 跳转 /connect/logout
```

## Admin 在 Identity 中的注册信息

```json
{
  "ClientId": "admin_client",
  "ClientSecret": "admin-secret-key",
  "ClientType": "confidential",
  "GrantTypes": ["authorization_code", "refresh_token"],
  "RedirectUris": ["https://admin.domain.com/callback"],
  "PostLogoutRedirectUris": ["https://admin.domain.com"],
  "Scopes": ["openid", "profile", "roles", "api"]
}
```

## 角色设计

| 角色 | 权限范围 | 说明 |
|------|---------|------|
| **SuperAdmin** | 全部权限 | 系统管理员，可管理一切 |
| **Operator** | 运维操作 | 爬虫启停、ETL 触发、重试、查看日志 |
| **DataEditor** | 数据编辑 | 品牌/类型/轴承/商家 CRUD、映射维护、图片修正 |
| **Viewer** | 只读 | 查看所有页面，无任何写操作 |

## Identity API 端点要求

Admin 通过 Identity API 管理用户和角色。需要 Identity 暴露以下端点：

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/users` | 用户列表 |
| GET | `/api/users/{id}` | 用户详情 |
| POST | `/api/users` | 创建用户 |
| PUT | `/api/users/{id}` | 更新用户 |
| DELETE | `/api/users/{id}` | 删除用户 |
| GET | `/api/roles` | 角色列表 |
| POST | `/api/roles` | 创建角色 |
| PUT | `/api/roles/{id}` | 更新角色 |
| DELETE | `/api/roles/{id}` | 删除角色 |
| GET | `/live` | 存活探针 |

OpenFindBearings.Identity 当前未实现这些管理端点，需要新增。
