# 配置管理模块 v1.0.0

## 概述

配置管理模块统一管理各项目的运行时配置参数，并通过各项目的 API 进行修改。配置变更可能需要重新部署 K3s ConfigMap 才能生效，Admin 负责展示配置差异和修改指引。

## 配置源

| 项目 | 配置形式 | 修改方式 | 生效条件 |
|------|---------|---------|---------|
| Crawler | appsettings.json / ConfigMap | API | 需重启 Pod |
| Sync | appsettings.json / ConfigMap | API | 需重启 Pod |
| API | appsettings.json / ConfigMap | API | 需重启 Pod |
| Identity | appsettings.json / ConfigMap | API | 需重启 Pod |
| Admin | appsettings.json / ConfigMap | API | 需重启 Pod |
| CronJob 调度 | CronJob YAML | kubectl apply | 下次调度生效 |

## 配置持久化策略

由于 ConfigMap 不会随 Pod 重启自动更新，Admin 的配置修改采用以下流程：

```
管理员修改配置 → 生成新的 ConfigMap YAML 片段
              → 提示管理员执行 kubectl apply
              → 滚动重启 Deployment
```

也可以提供一键脚本：

```bash
# Admin 生成 ConfigMap YAML 并直接执行 kubectl
# 前提: Admin Pod 有 kubectl 权限（RBAC 配置）
```
