## 1. 概述

Admin 项目通过标准 JWT Bearer 认证接入 Identity 认证中心，实现用户登录、权限控制和本地 RBAC 管理。

### 1.1 认证架构

```
用户浏览器
    │
    ▼
Admin 项目（JWT Bearer + Cookie）
    │
    ├── 未登录 → 跳转 Identity /connect/authorize → 登录 → 回调拿 code → 换 JWT
    ├── 已登录 → JWT 验证通过 → 本地 RBAC 权限校验
    │
    ▼
API 项目（JWT 验证 + PermissionFilter）
    │
    └── 验证 JWT 有效性 → 校验具体权限
```

### 1.2 技术选型

| 组件 | 选型 | 说明 |
|------|------|------|
| 认证方案 | JWT Bearer + Cookie | 标准 JWT 验证，不用 OpenIddict 客户端包 |
| 认证中心 | Identity（OpenIddict） | OIDC 授权码流程 |
| 会话管理 | Cookie | 存储 JWT access_token |
| 权限控制 | 本地 RBAC（db_admin） | admin_role_permissions 表 |

## 2. OIDC 认证流程

### 2.1 授权码流程

```
1. 用户访问 Admin 任意页面
2. [Authorize] 未认证 → 401 → 跳转 /Account/Login
3. /Account/Login → 构造 OIDC 请求 → 跳转 Identity /connect/authorize
4. Identity 检查登录状态 → 未登录 → 登录页
5. 用户登录 → Identity 授予 code → 回调 Admin /Account/Callback?code=xxx
6. Admin 用 code + client_secret POST 到 Identity /connect/token
7. 拿到 access_token + refresh_token
8. access_token 存 Cookie → 跳转原始请求页面
```

### 2.2 Token 刷新（无感）

```
1. JWT 过期 → API 返回 401
2. Admin 检查 Cookie 中的 refresh_token
3. 用 refresh_token POST 到 Identity /connect/token
4. 获取新 access_token + refresh_token
5. 更新 Cookie → 重试原始请求
6. refresh_token 也过期 → 跳转重新登录
```

### 2.3 登出流程

```
1. 用户点击退出
2. /Account/Logout → 清除本地 Cookie
3. 跳转 Identity /connect/logout
4. Identity 清除会话 → 跳转 Admin 首页
```

## 3. 端口与回调配置

### 3.1 开发环境

| 项目 | 端口 | URL |
|------|------|-----|
| Identity | HTTPS 7201 | `https://localhost:7201` |
| Admin | HTTPS 7167 | `https://localhost:7167` |
| API | HTTPS 7183 | `https://localhost:7183` |

### 3.2 回调地址

| 配置项 | 值 |
|--------|-----|
| Admin CallbackPath | `/callback` |
| Identity redirect_uri | `https://localhost:7167/callback` |
| Identity PostLogoutRedirectUri | `https://localhost:7167/` |

### 3.3 生产环境回调地址维护

OpenIddict 的 redirect_uri 存储在数据库 `OpenIddictApplications` 表中。维护方式：

| 场景 | 操作 |
|------|------|
| 首次部署 | 更新 SeedData 或通过 ApplicationController 端点更新 |
| 域名变更 | 调用 `PUT /api/applications/{id}` 更新 redirect_uris |
| 多环境共存 | 同一 ClientId 可注册多个 redirect_uri |

## 4. NuGet 包

| 包名 | 用途 |
|------|------|
| `Microsoft.AspNetCore.Authentication.JwtBearer` | JWT Bearer 认证 |
| `Microsoft.AspNetCore.Authentication.Cookies` | Cookie 会话管理 |

**不使用** `OpenIddict.AspNetCore` — Admin 作为纯 JWT 客户端，通过 HTTP 调用 Identity 端点。

## 5. 认证中间件配置

```csharp
// Program.cs
builder.Services.AddAuthentication(CookieAuthenticationDefaults.AuthenticationScheme)
    .AddCookie(options =>
    {
        options.LoginPath = "/Account/Login";
        options.LogoutPath = "/Account/Logout";
        options.ExpireTimeSpan = TimeSpan.FromHours(8);
    })
    .AddJwtBearer(options =>
    {
        options.Authority = "https://localhost:7201";
        options.Audience = "admin_client";
        options.RequireHttpsMetadata = false;
        options.SaveTokens = true;
    });
```

## 7. 用户管理（调 Identity API）

### 7.1 数据流

```
Admin UsersController
    │
    ├── GET /api/Account/admin/users?tenantId=xxx → Identity 用户列表
    ├── POST /api/Account/admin/users → Identity 创建用户
    ├── PUT /api/Account/admin/users/{id} → Identity 更新用户
    ├── POST /api/Account/admin/users/{id}/reset-password → Identity 重置密码
    ├── PATCH /api/Account/admin/users/{id}/status → Identity 启用/禁用
    ├── DELETE /api/Account/admin/users/{id} → Identity 软删除
    ├── POST /api/Account/admin/users/{id}/restore → Identity 恢复
    │
    └── 本地 db_admin
        ├── SELECT admin_user_roles WHERE UserId = @id → 获取角色
        ├── INSERT admin_user_roles → 分配角色
        ├── UPDATE admin_user_roles → 变更角色
        └── DELETE admin_user_roles WHERE UserId = @id → 清理角色
```

### 7.2 页面功能

| 功能 | 说明 |
|------|------|
| 用户列表 | 搜索 + 分页 + 状态筛选 + 角色标记（已接入/未接入） |
| 创建用户 | 用户名 + 邮箱 + 密码 + 角色选择（可选） |
| 编辑用户 | 修改用户名/邮箱/手机号 |
| 重置密码 | 输入新密码 |
| 启用/禁用 | 切换用户状态 |
| 分配角色 | 选择 Admin 业务角色（admin/业务员/审计员/自定义） |
| 删除/恢复 | 软删除 + 恢复已删除用户 |

## 11. 与 Identity 的边界

| 操作 | Admin 处理 | Identity 处理 |
|------|-----------|--------------|
| 用户认证 | Cookie 存储 JWT | OIDC 签发 token |
| 用户 CRUD | HTTP 调用 Identity API | 实际数据库操作 |
| 角色定义 | 本地 admin_role_permissions | 系统级 ASP.NET roles |
| 权限校验 | 本地 if/else | 不涉及 |
| 租户筛选 | 传 tenantId 参数 | 数据库过滤 |
