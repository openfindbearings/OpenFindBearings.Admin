# 部署与降级策略 v1.0.0

## 概述

Admin 作为统一管理后台，部署在 K3s 集群中。由于 Admin 依赖 4 个后端服务，但我们设计了优雅降级策略——任一服务离线时 Admin 仍可正常启动和导航，仅受影响的功能禁用。

## 服务依赖拓扑

```
Admin 启动
  │
  ├─ 读取本地配置（无依赖）
  ├─ 检查 /live 探针
  │
  ├─ 仪表盘 → 全部服务（降级: 有数据就显示，无数据提示离线）
  ├─ Crawler 页面 → Crawler API（离线: 禁用启停，展示上次缓存状态）
  ├─ Sync 页面 → Sync API（离线: 禁用ETL操作，展示缓存日志）
  ├─ Data 页面 → API（离线: 显示"服务不可用"）
  ├─ Mapping 页面 → API + Crawler API（离线: 部分可用）
  ├─ Image 页面 → API（离线: 不可用）
  ├─ Config 页面 → 全部服务（离线: 显示"不可修改"）
  └─ Identity 页面 → Identity API（离线: 不可用）
```

## 健康检测实现

```csharp
// Services/ServiceHealthService.cs
public class ServiceHealthService : BackgroundService
{
    private readonly ConcurrentDictionary<string, ServiceStatus> _statuses = new();
    private readonly IHttpClientFactory _httpClientFactory;
    private readonly ILogger<ServiceHealthService> _logger;
    private readonly IServiceProvider _serviceProvider;

    public record ServiceStatus(string Name, bool IsOnline, DateTime LastChecked, string? Error);

    public event Action? OnStatusChanged;  // SignalR 推送

    public ServiceStatus GetStatus(string serviceName) =>
        _statuses.GetValueOrDefault(serviceName, new ServiceStatus(serviceName, false, DateTime.MinValue, "未检测"));

    public bool IsOnline(string serviceName) => GetStatus(serviceName).IsOnline;

    protected override async Task ExecuteAsync(CancellationToken ct)
    {
        while (!ct.IsCancellationRequested)
        {
            await CheckServiceAsync("Crawler");
            await CheckServiceAsync("Sync");
            await CheckServiceAsync("Api");
            await CheckServiceAsync("Identity");

            OnStatusChanged?.Invoke();
            await Task.Delay(TimeSpan.FromSeconds(15), ct);
        }
    }

    private async Task CheckServiceAsync(string serviceName)
    {
        var config = _serviceProvider.GetRequiredService<IConfiguration>();
        var baseUrl = config[$"Services:{serviceName}:BaseUrl"];

        try
        {
            using var cts = new CancellationTokenSource(TimeSpan.FromSeconds(5));
            var client = _httpClientFactory.CreateClient("HealthCheck");
            var response = await client.GetAsync($"{baseUrl}/live", cts.Token);

            _statuses[serviceName] = new ServiceStatus(
                serviceName, response.IsSuccessStatusCode, DateTime.UtcNow, null);
        }
        catch (Exception ex)
        {
            _logger.LogWarning("服务 {Name} 离线: {Error}", serviceName, ex.Message);
            _statuses[serviceName] = new ServiceStatus(
                serviceName, false, DateTime.UtcNow, ex.Message);
        }
    }
}
```

public class StatusHub : Hub

    }

```javascript
// wwwroot/js/site.js
const connection = new signalR.HubConnectionBuilder()
    .withUrl("/hubs/status")
    .build();

connection.on("StatusChanged", (statuses) => {
    statuses.forEach(s => {
        const el = document.getElementById(`status-${s.name}`);
        if (el) {
            el.className = s.isOnline ? "status-online" : "status-offline";
            el.title = s.isOnline ? `${s.name} 运行中` : `${s.name} 离线`;
        }
    });
});

connection.start();
```

## 降级策略矩阵

| 服务离线 | Admin 受影响功能 | 用户看到 |
|---------|----------------|---------|
| Crawler | 爬虫管理全部 | 状态条显示 🔴，按钮禁用，提示"Crawler 服务不可用" |
| Sync | 同步管理全部 | 同 Crawler |
| API | 数据管理、映射、图片 | 页面显示"API 服务不可用"，无法查看/编辑业务数据 |
| Identity | 登录、用户管理 | 已登录用户继续用（Cookie 有效期内），新用户无法登录 |
| 全部离线 | 全部功能 | 仅 Admin 首页正常，导航链接全禁用 |

## 启动顺序

```
首次部署优先：
  1. Identity（认证基础，必须先启动）
  2. API（业务数据基础）
  3. Sync（依赖 API）
  4. Crawler（独立运行）
  5. Admin（最后启动，不影响其他服务）
```

Admin 的设计保证了启动不依赖任何服务——没有强制等待、没有启动失败连锁。
