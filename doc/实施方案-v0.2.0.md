
## 二、Admin 数据库（db_admin）

4 张表，PostgreSQL，EF Core + Npgsql：

| 表 | 用途 | 关键字段 |
|---|---|---|
| `admin_audit_logs` | Admin 操作审计 | Id, UserId, Username, Action, ResourceType, ResourceId, Detail, Result, IpAddress, CreatedAt |
| `admin_user_roles` | Admin 角色分配 | Id, UserId, RoleName, AssignedAt |
| `admin_role_permissions` | 角色权限映射 | Id, RoleName, PermissionKey, Granted, CreatedAt |
| `admin_configs` | Admin 本地配置 | Id, Key(unique), Value, Description, UpdatedAt |

种子数据：admin/editor/viewer 三个角色的权限映射（17 个权限键，使用 Bearing* 命名）。

## 三、各页面功能与 API 对接

### 3.1 品牌列表

| 操作 | API 端点 | 说明 |
|------|---------|------|
| 列表 | `GET /api/brands` | 分页，支持 includeDeleted |
| 创建 | `POST /api/admin/brands` | Code(必填), Name(必填), Country?, Level? |
| 编辑 | `PUT /api/admin/brands/{id}` | Name?, Country?, Level?（Code 不可改） |
| 删除 | `DELETE /api/admin/brands/{id}` | 软删除（IsActive=false） |
| 恢复 | `PUT /api/admin/brands/{id}/restore` | 恢复已删除品牌 |
| 彻底删除 | `DELETE /api/admin/brands/{id}/hard` | 需 DataRestore 权限，仅超级管理员 |

### 3.2 类型列表

| 操作 | API 端点 | 说明 |
|------|---------|------|
| 列表 | `GET /api/bearing-types` | 分页，支持 includeDeleted |
| 创建 | `POST /api/admin/bearing-types` | Code(必填), Name(必填), Description? |
| 编辑 | `PUT /api/admin/bearing-types/{id}` | Name?, Description? |
| 删除 | `DELETE /api/admin/bearing-types/{id}` | 软删除 |
| 恢复 | `PUT /api/admin/bearing-types/{id}/restore` | 恢复已删除类型 |
| 彻底删除 | `DELETE /api/admin/bearing-types/{id}/hard` | 需 DataRestore 权限 |

## 五、RBAC 架构

各系统自己维护 RBAC，Identity 只管 OIDC + 系统级 scope：

```
Identity
  ├ 注册 admin_client（scope: admin）
  ├ 注册 mobile_client（scope: mobile）
  └ 注册 sync_client（scope: sync）

Admin 项目
  ├ db_admin.admin_role_permissions（本地角色→权限映射）
  ├ 每次请求从 db_admin 加载当前用户的权限列表
  ├ PermissionKey 枚举使用 Bearing* 命名（17 个权限键）
  └ 3 个预置角色：admin（全部权限）、editor（业务员）、viewer（审计员）

其他系统（Mobile 等）
  └ 各自维护自己的角色权限，模式同 Admin
```
